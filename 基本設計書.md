# 基本設計書

## 1. システム概要

本システムは、ユーザーのログイン／ログアウト機能および管理者によるユーザー管理機能を提供するWebアプリケーションである。

ユーザーはメールアドレスとパスワードを用いてログインし、ログイン状態および権限に応じて利用可能な画面・操作を制御する。

認証処理にはDeviseを使用し、ログイン状態の管理を行う。  
認可処理にはCanCanCanを使用し、管理者ユーザーとゲストユーザーの操作権限を制御する。

## 2. 対象ユーザー

| 区分 | 概要 |
|---|---|
| 未ログインユーザー | ログインしていないユーザー。ログイン画面のみ利用可能。 |
| ゲストユーザー | 一般利用者。ログイン後、自身の情報を確認できる。 |
| 管理者ユーザー | ユーザー管理機能を利用できるユーザー。 |

## 6. 画面遷移

### 6.1 未ログイン時

```text
未ログインユーザー
  ↓
ログイン画面
  ↓ ログイン成功
ログイン後の遷移先へ
```

未ログイン状態で認証が必要な画面へアクセスした場合、ログイン画面へ遷移する。

### 6.2 ログイン後

```text
ゲストユーザー
  ↓
マイページ

管理者ユーザー
  ↓
ユーザー一覧画面
```

ログイン後の遷移先は、ユーザーの権限によって分岐する。

| 権限 | ログイン後の遷移先 |
|---|---|
| ゲストユーザー | マイページ |
| 管理者ユーザー | ユーザー一覧画面 |

### 6.3 ログアウト時

```text
ログイン中ユーザー
  ↓ ログアウト
ログイン画面
```

ログアウト後はログイン画面へ遷移する。

## 7. 認証・認可設計

## 7.1 認証方針

本システムでは、ログイン状態の判定にDeviseを使用する。

Deviseにより、以下の処理を行う。

| 処理 | 内容 |
|---|---|
| ログイン | 登録済みメールアドレスとパスワードを用いて認証する |
| ログアウト | ログイン状態を解除する |
| ログイン状態判定 | ユーザーがログイン中かどうかを判定する |
| 未ログイン時制御 | 認証が必要な画面へ未ログイン状態でアクセスした場合、ログイン画面へ遷移させる |

## 7.2 認可方針

本システムでは、操作権限の判定にCanCanCanを使用する。

CanCanCanにより、以下の制御を行う。

| 権限 | 制御内容 |
|---|---|
| 管理者ユーザー | ユーザー管理機能を利用できる |
| ゲストユーザー | ユーザー管理機能を利用できない |
| 未ログインユーザー | 認証が必要な画面へアクセスできない |

権限定義はCanCanCanのAbilityで管理する。  
管理者ユーザーとゲストユーザーの区分に基づき、利用可能な画面・操作を制御する。

## 8. 権限設計

| 操作 | 未ログインユーザー | ゲストユーザー | 管理者ユーザー |
|---|---:|---:|---:|
| ログイン | ○ | × | × |
| ログアウト | × | ○ | ○ |
| マイページ閲覧 | × | ○ | ○ |
| ユーザー一覧閲覧 | × | × | ○ |
| ユーザー新規登録 | × | × | ○ |
| ユーザー編集 | × | × | ○ |
| ユーザー削除 | × | × | ○ |
| 権限変更 | × | × | ○ |

## 9. データ設計概要

### 9.1 User

| 項目名 | 概要 |
|---|---|
| ID | ユーザーを識別するためのID |
| 名前 | ユーザーの表示名 |
| メールアドレス | ログイン時に使用するメールアドレス |
| パスワード | ログイン時に使用するパスワード |
| 権限 | ゲストユーザーまたは管理者ユーザーを区分する |

### 9.2 パスワード管理

パスワードは平文では保存せず、Deviseの仕組みによりハッシュ化された状態で保存する。

### 9.3 権限情報

ユーザーの権限はUser情報の一部として管理する。  
権限には、ゲストユーザーと管理者ユーザーを設定する。

| 権限 | 内容 |
|---|---|
| guest | 一般利用者。自身の情報確認のみ可能。 |
| admin | 管理者。ユーザー管理機能を利用可能。 |

## 10. 機能別設計

## 10.1 ログイン機能

### 概要

登録済みユーザーが、メールアドレスとパスワードを入力してログインできる機能である。

### 入力項目

| 項目 | 必須 | 内容 |
|---|---|---|
| メールアドレス | 必須 | 登録済みのメールアドレス |
| パスワード | 必須 | 登録済みのパスワード |

### 処理概要

1. ユーザーはログイン画面を表示する。
2. メールアドレスとパスワードを入力する。
3. 入力内容を送信する。
4. Deviseにより認証処理を行う。
5. 認証に成功した場合、ログイン状態にする。
6. 認証に失敗した場合、ログイン画面にエラーメッセージを表示する。

### 正常時の動作

| 権限 | 遷移先 |
|---|---|
| ゲストユーザー | ゲストページ |
| 管理者ユーザー | 管理者ページ |

### 異常時の動作

| ケース | 動作 |
|---|---|
| メールアドレスが未入力 | エラーメッセージを表示する |
| パスワードが未入力 | エラーメッセージを表示する |
| 認証情報が一致しない | エラーメッセージを表示する |

## 10.2 ログアウト機能

### 概要

ログイン中のユーザーが、ログイン状態を解除できる機能である。

### 処理概要

1. ログイン中のユーザーがログアウト操作を行う。
2. Deviseによりログイン状態を解除する。
3. ログアウト後、ログイン画面へ遷移する。

## 10.3 アクセス制御機能

### 概要

ログイン状態や権限に応じて、利用可能な画面・操作を制限する機能である。

認証判定にはDeviseを使用し、権限判定にはCanCanCanを使用する。

### 制御内容

| 条件 | 動作 |
|---|---|
| 未ログインユーザーがログイン必須画面へアクセス | ログイン画面へ遷移する |
| ゲストユーザーが管理者用画面へアクセス | アクセスを拒否し、マイページまたはトップ画面へ遷移する |
| 管理者ユーザーが管理者用画面へアクセス | 画面を表示する |

## 10.4 ユーザー一覧表示機能

### 概要

管理者ユーザーが、登録済みユーザーを一覧で確認できる機能である。

### 表示項目

| 項目 | 内容 |
|---|---|
| ID | ユーザーID |
| 名前 | ユーザー名 |
| メールアドレス | 登録メールアドレス |
| 権限 | ゲストまたは管理者 |
| 作成日時 | ユーザー登録日時 |

### 処理概要

1. 管理者ユーザーがユーザー一覧画面へアクセスする。
2. CanCanCanにより、ユーザー一覧閲覧権限を確認する。
3. 権限がある場合、登録済みユーザー情報を取得する。
4. ユーザー一覧を画面に表示する。
5. 権限がない場合、アクセスを拒否する。

## 10.5 ユーザー詳細表示機能

### 概要

管理者ユーザーが、選択したユーザーの詳細情報を確認できる機能である。

### 表示項目

| 項目 | 内容 |
|---|---|
| ID | ユーザーID |
| 名前 | ユーザー名 |
| メールアドレス | 登録メールアドレス |
| 権限 | ゲストまたは管理者 |
| 作成日時 | ユーザー登録日時 |
| 更新日時 | ユーザー情報の最終更新日時 |

### 処理概要

1. 管理者ユーザーがユーザー詳細画面へアクセスする。
2. CanCanCanにより、ユーザー詳細閲覧権限を確認する。
3. 権限がある場合、対象ユーザー情報を取得する。
4. ユーザー詳細を画面に表示する。
5. 権限がない場合、アクセスを拒否する。

## 10.6 ユーザー登録機能

### 概要

管理者ユーザーが、新規ユーザーを登録できる機能である。

### 入力項目

| 項目 | 必須 | 内容 |
|---|---|---|
| 名前 | 必須 | ユーザーの表示名 |
| メールアドレス | 必須 | ログインに使用するメールアドレス |
| パスワード | 必須 | ログインに使用するパスワード |
| 権限 | 必須 | ゲストまたは管理者 |

### 処理概要

1. 管理者ユーザーがユーザー新規登録画面を表示する。
2. CanCanCanにより、ユーザー登録権限を確認する。
3. 必要項目を入力する。
4. 登録操作を行う。
5. 入力内容に問題がなければ、ユーザーを登録する。
6. 入力内容に問題がある場合、エラーメッセージを表示する。

## 10.7 ユーザー編集機能

### 概要

管理者ユーザーが、既存ユーザーの情報を編集できる機能である。

### 編集可能項目

| 項目 | 内容 |
|---|---|
| 名前 | ユーザーの表示名 |
| メールアドレス | ログインに使用するメールアドレス |
| 権限 | ゲストまたは管理者 |

### 処理概要

1. 管理者ユーザーがユーザー編集画面を表示する。
2. CanCanCanにより、ユーザー編集権限を確認する。
3. 編集内容を入力する。
4. 更新操作を行う。
5. 入力内容に問題がなければ、ユーザー情報を更新する。
6. 入力内容に問題がある場合、エラーメッセージを表示する。

## 10.8 ユーザー削除機能

### 概要

管理者ユーザーが、不要なユーザーを削除できる機能である。

### 処理概要

1. 管理者ユーザーが削除対象ユーザーを選択する。
2. CanCanCanにより、ユーザー削除権限を確認する。
3. 権限がある場合、削除操作を行う。
4. 確認後、対象ユーザーを削除する。
5. 削除後、ユーザー一覧画面へ遷移する。

### 制約

管理者ユーザーが自分自身を削除できるかどうかは、運用方針に応じて決定する。  
初期実装では、誤操作防止のため、管理者自身の削除は不可とする。

## 11. バリデーション概要

| 対象 | 項目 | 内容 |
|---|---|---|
| User | 名前 | 未入力を許可しない |
| User | メールアドレス | 未入力を許可しない |
| User | メールアドレス | 重複を許可しない |
| User | パスワード | 未入力を許可しない |
| User | 権限 | ゲストまたは管理者のいずれかを設定する |

## 12. エラー・例外時の挙動

| ケース | 動作 |
|---|---|
| ログイン失敗 | ログイン画面にエラーメッセージを表示する |
| 未ログインで認証必須画面へアクセス | ログイン画面へ遷移する |
| ゲストユーザーが管理者画面へアクセス | アクセスを拒否し、マイページへ遷移する |
| ユーザー登録時の入力不備 | 登録画面にエラーメッセージを表示する |
| ユーザー編集時の入力不備 | 編集画面にエラーメッセージを表示する |
| 存在しないユーザーへアクセス | エラー画面または一覧画面へ遷移する |
| CanCanCanによる権限エラー | アクセスを拒否し、適切な画面へ遷移する |

## 13. セキュリティ方針

| 項目 | 方針 |
|---|---|
| パスワード管理 | パスワードは平文保存せず、ハッシュ化して保存する |
| 認証制御 | Deviseを使用し、ログインしていないユーザーは認証必須画面へアクセスできないようにする |
| 権限制御 | CanCanCanを使用し、管理者以外はユーザー管理機能を利用できないようにする |
| エラー表示 | 認証失敗時に、どの項目が誤っているかを過度に明示しない |

## 14. 補足

本設計書では、ログイン／ログアウト機能およびユーザー管理機能の基本的な画面構成、権限、処理概要を定義する。

具体的なルーティング、コントローラー処理、モデル定義、マイグレーション内容、CanCanCanのAbility定義、バリデーション実装については、詳細設計書で定義する。
