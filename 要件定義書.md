| 項目 | 内容 |
|---|---|

| 作成日 |  |

---

## 2. 背景・課題

### 背景

研修課題として、認証機能、権限制御、ユーザー管理機能を持つ基本的なWebシステムを作成する。

### 課題

| No | 課題内容 |
|---|---|
| 1 | ユーザーが安全にログイン・ログアウトできる仕組みが必要である |
| 2 | 一般ユーザーとアドミンユーザーを区分する必要がある |
| 3 | アドミンユーザーがユーザー情報を管理できる必要がある |
| 4 | 未ログインユーザーや一般ユーザーが管理機能にアクセスできないようにする必要がある |

---

## 3. 目的

本システムでは、以下を実現することを目的とする。

- 登録済みユーザーがログイン・ログアウトできる
- パスワードを安全に扱える
- ログイン状態に応じてアクセス制御ができる
- 一般ユーザーとアドミンユーザーを区分できる
- アドミンユーザーがユーザー情報を作成・閲覧・更新・削除できる

---

## 6. 利用者

| 利用者区分 | 説明 |
|---|---|
| 一般ユーザー | ログイン・ログアウトができるユーザー。管理機能は利用できない |
| アドミンユーザー | ログイン・ログアウトに加えて、ユーザー管理機能を利用できるユーザー |
| 未ログインユーザー | ログイン前のユーザー。認証が必要な画面にはアクセスできない |

---

## 7. 機能要件

### 7.1 認証機能

| ID | 機能名 | 要件 | 優先度 |
|---|---|---|---|
| F-001 | ログイン | 登録済みユーザーはログインできること | 高 |
| F-002 | ログアウト | ログイン中のユーザーはログアウトできること | 高 |
| F-003 | パスワード管理 | パスワードを安全に保存できること | 高 |
| F-004 | ログイン状態管理 | ログイン中のユーザーを判定できること | 高 |
| F-005 | ログイン失敗制御 | 一定回数ログインに失敗した場合、ログイン試行を制限できること | 高 |
| F-006 | 未ログイン時アクセス制御 | 未ログインユーザーは認証が必要な画面にアクセスできないこと | 高 |

### 7.2 ユーザー管理機能

| ID | 機能名 | 要件 | 優先度 |
|---|---|---|---|
| F-101 | ユーザー一覧 | アドミンユーザーは登録済みユーザーの一覧を確認できること | 高 |
| F-102 | ユーザー作成 | アドミンユーザーはユーザーを新規作成できること | 高 |
| F-103 | ユーザー詳細 | アドミンユーザーはユーザーの詳細情報を確認できること | 高 |
| F-104 | ユーザー編集 | アドミンユーザーはユーザー情報を編集できること | 高 |
| F-105 | ユーザー削除 | アドミンユーザーはユーザーを削除できること | 高 |
| F-106 | ユーザー区分 | ユーザーを一般ユーザーとアドミンユーザーに区分できること | 高 |
| F-107 | 入力バリデーション | 必須項目や入力形式に対するバリデーションがあること | 高 |
| F-108 | 重複チェック | メールアドレスなど一意であるべき項目の重複を防止できること | 高 |

### 7.3 権限制御機能

| ID | 機能名 | 要件 | 優先度 |
|---|---|---|---|
| F-201 | 管理画面アクセス制御 | アドミンユーザーのみ管理画面にアクセスできること | 高 |
| F-202 | 一般ユーザー制御 | 一般ユーザーはユーザー管理機能を利用できないこと | 高 |
| F-203 | 未ログインユーザー制御 | 未ログインユーザーはログインが必要な画面にアクセスできないこと | 高 |

---

## 8. 非機能要件

| 分類 | 要件内容 |
|---|---|
| セキュリティ | パスワードは平文で保存しない |
| セキュリティ | 未ログインユーザーは認証が必要な画面へアクセスできない |
| セキュリティ | 一般ユーザーはアドミン用の管理画面へアクセスできない |
| 操作性 | ログイン、ログアウト、ユーザー管理の操作が画面上から行える |
| 保守性 | 認証機能、ユーザー管理機能、権限制御の役割を分けて実装する |
| 信頼性 | 入力エラー時には登録・更新を行わず、エラー内容を表示する |

---

|---|---|---|---|

| 一般画面閲覧 | 不可 | 可 | 可 |
| 管理画面閲覧 | 不可 | 不可 | 可 |
| ユーザー一覧閲覧 | 不可 | 不可 | 可 |
| ユーザー作成 | 不可 | 不可 | 可 |
| ユーザー詳細閲覧 | 不可 | 不可 | 可 |
| ユーザー編集 | 不可 | 不可 | 可 |
| ユーザー削除 | 不可 | 不可 | 可 |

## 11. データ要件

| データ名 | 内容 |
|---|---|
| ユーザーID | ユーザーを一意に識別するID |
| メールアドレス | ログインに使用する情報。重複不可 |
| パスワード | ログインに使用する情報。安全な形式で保存する |
| ユーザー区分 | 一般ユーザーまたはアドミンユーザーを判定する情報 |
| 作成日時 | ユーザー情報が作成された日時 |
| 更新日時 | ユーザー情報が更新された日時 |

## 12. 制約事項

| No | 制約内容 |
|---|---|
| 1 | 研修課題として必要最低限の機能に絞って開発する |
| 2 | 対象外に記載した機能は今回の開発では実装しない |
| 3 | 認証処理には安全性を考慮し、パスワードを平文保存しない |
| 4 | アドミン権限が必要な処理は、画面表示だけでなく処理側でも制御する |
